

# 自举原理初探

编程语言的自举（Bootstrapping）是指一种编程语言能够编译自身编译器的过程。简单来说，就是：先用其他语言（或该语言的早期版本）编写一个基础编译器，再用该语言本身编写更完善的编译器源码，最后用基础编译器编译这个源码，得到可编译自身的 “自举编译器”。

**自举的核心步骤**，这里以C语言和mini语言为例: **阶段1**：用C语言编写mini语言的基础编译器（能编译mini语言的核心语法） **阶段2**：用语言 B 编写语言 B 的完整编译器源码（功能与阶段 1 的编译器等效，甚至更强）

```bash
gcc  mini_print.c  -o  mini_print.elf

gcc  test_print.c && ./a.out
```


mini_print.c 编译源代码如下：
```cpp
#include <stdio.h>
#include <string.h>
#include <stdlib.h>

// 核心功能：将MiniLang代码转换为C代码
void mini_to_c(const char* input_file, const char* output_c) {
    FILE *in = fopen(input_file, "r");
    FILE *out = fopen(output_c, "w");
    if (!in || !out) { perror("文件打开失败"); exit(1); }

    // 生成C代码的头部
    fprintf(out, "#include <stdio.h>\n");
    fprintf(out, "int main() {\n");

    char line[1024];
    while (fgets(line, sizeof(line), in)) {
        // 解析MiniLang的print语句：提取双引号 "" 中的内容
        char* print_keyword = strstr(line, "print ");
        if (print_keyword) {
            char* start = strchr(line, '"');
            char* end = start ? strrchr(line, '"') : NULL;
            if (start && end && start < end) {
                // 提取字符串内容，生成printf语句
                *end = '\0'; // 截断末尾的"
                fprintf(out, "    printf(\"%s\\n\");\n", start + 1);
            }
        }
    }

    // 生成C代码的尾部
    fprintf(out, "    return 0;\n");
    fprintf(out, "}\n");

    fclose(in);
    fclose(out);
}

int main(int argc, char* argv[]) {
    if (argc != 3) {
        printf("用法：%s <input.mini> <output.c>\n", argv[0]);
        return 1;
    }
    const char *input_mini = argv[1];
    const char *output_c = argv[2];

    // 1. 将MiniLang代码转换为C代码
    mini_to_c(input_mini, output_c);

    printf("转换完成：生成C代码文件 %s\n", output_c);
    return 0;
}
```

## 闭包

尤其编程语言的条件语句块与循环语句块互相嵌套的时候，这时候的代码词法分析以及语义分析就非常需要闭包这种能力了，因为常规的处理会非常的棘手。这时候才明白了闭包出现的原因，至少在编译器领域，它能解决这种逐渐深入逐渐展开最后又逐渐收拢的结构。

闭包的核心思想：让函数能够携带自己的执行环境。体现了闭包 "携带状态" 的核心特性。



# 移动语义

**提问豆包**: 用C语言写一个最小编译器，用来实现移动语义

**豆包回答**: 要实现一个支持 “移动语义” 的最小编译器，我们需要设计一个极简的编程语言（包含移动操作语法），并通过编译器将其转换为 C 代码（利用之前模拟移动语义的方式）。


**整体设计**:

- 定义语法；
- 编译器功能：
    - 词法分析：将源代码解析为 token（关键字、标识符、符号等）
    - 语法分析：构建抽象语法树（AST），识别移动操作
    - 代码生成：将 AST 转换为 C 代码，用`buffer_move`函数实现移动语义


**步骤1：定义mini语法**

支持以下语法:
```
// 声明Buffer变量（大小为数字）
Buffer 变量名(大小);

// 普通赋值（深拷贝）
变量名 = 变量名;

// 移动操作（资源转移）
变量名 = move(变量名);

// 写入数据
变量名.write("字符串");

// 打印变量状态
print(变量名);
```



示例程序（test.mini）：
```
Buffer a(100);    // 声明a，大小100
a.write("aaaaa"); // 写入数据
print(a);         // 打印a

Buffer b(50);     // 声明b
b = move(a);      // 移动a到b（a失去资源）
print(a);         // 打印a（应显示无资源）
print(b);         // 打印b（应显示a的数据）

Buffer c(10);
c = b;            // 普通赋值（深拷贝）：c = b;
print(c);         // 打印c（应显示a的数据）
print(b);         // 打印b（应显示a的数据）
```


下面是用于测试的无注释版代码(因为当前实现的编译器不支持注释语句的解析)：
```
Buffer a(100);
a.write("aaaaa");
print(a);

Buffer b(50);
b = move(a);
print(a);
print(b);

Buffer c(10);
c = b;
print(c);
print(b);
```

**步骤3：使用编译器**
下面是编译器使用流程：

```bash
#先把编译器编译出来
gcc mini_move.c -o mini_move.elf

#然后用编译器去编译测试代码
./mini_move.elf test_move.mini test_move.c

#把最终代码再次编译成可执行程序
gcc test_move.c && ./a.out
```


**核心原理说明：**

1. **移动语义的实现**：编译器将源语言的`b = move(a)`转换为 C 代码的`buffer_move(&b, &a)`，实现资源所有权转移（`a`的`data`指针被置空，`b`接管内存）。
2. **与普通赋值的区别**：普通赋值（`b = a`）被转换为深拷贝（重新分配内存并复制内容），而移动操作仅转移指针，避免内存拷贝，体现 “移动语义” 的高效性。
3. **编译器流程**：
    - 词法分析：将源代码拆分为`Token`（如`Buffer`、`move`、标识符等）。
    - 语法分析：根据 Token 构建 AST，区分声明、赋值、移动等操作。
    - 代码生成：将 AST 转换为带`buffer_move`等函数的 C 代码，完成移动语义的模拟。

这个最小编译器展示了移动语义的核心思想（资源所有权转移），以及如何通过编译过程将高级语法转换为底层实现。实际编译器会更复杂，但核心原理一致。

Rust语言的赋值语句默认就是移动语义，如果开发者想深拷贝，就要明确使用clone这个trait。



# Rust编程语言

- 所有的程序都必须和计算机内存打交道，如何从内存中申请空间来存放程序的运行内容，如何在不需要的时候释放这些空间，成了重中之重，也是所有编程语言设计的难点之一。在计算机语言不断演变过程中，出现了三种流派：
    - **垃圾回收**：在程序运行时不断寻找不再使用的内存，典型代表：Java、Go。
    - **手动管理内存**：在程序中，通过函数调用的方式来申请和释放内存，典型代表：C/C++。
    - **通过所有权来管理内存**：编译器在编译时会根据一系列规则进行检查。

其中 Rust 选择了第三种，最妙的是，这种****检查只发生在编译期****，因此对于程序运行期，不会有任何性能上的损失。由于所有权是一个新概念，因此读者需要花费一些时间来掌握它，一旦掌握，海阔天空任你飞跃。
