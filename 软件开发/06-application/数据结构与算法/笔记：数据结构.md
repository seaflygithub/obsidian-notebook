## 基础二叉树和计算器

- 下面是最简单的二叉树，以及用二叉树实现了一个最简单的整数四则运算计算器。
```cpp {.line-numbers}
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <stdint.h>
#include <stdbool.h>
#include <stdarg.h>
#include <errno.h>
#include <ctype.h>
#include <unistd.h>

typedef struct Node {
	int data;
	struct Node *left;
	struct Node *right;
}Node;

Node *node_new(int data)
{
	Node *newptr = malloc(sizeof(Node));
	if (newptr == NULL)
	{
		fprintf(stderr, "malloc failed, %s\n", strerror(errno));
		return NULL;
	}

	newptr->data  = data;
	newptr->left  = NULL;
	newptr->right = NULL;
	return newptr;
}

bool is_leaf(Node *node)
{
	return ( (node) && (!node->left) && (!node->right) );
}

int tree_size(Node *root)
{
	if (!root) return 0;

	int ls = tree_size(root->left);
	int rs = tree_size(root->right);

	return (ls+rs+1);
}

int tree_leaves(Node *root)
{
	if (!root) return 0;
	if (is_leaf(root)) return 1;

	return tree_leaves(root->left) + tree_leaves(root->right);
}


typedef struct Calculator {
	char opdata[16];

	struct Calculator *tree_l;
	struct Calculator *tree_r;
}Calculator;

bool cal_is_leaf(Calculator *node)
{
	return ( (node) && (!node->tree_l) && (!node->tree_r) );
}

bool cal_is_num(Calculator *node)
{
    char *endptr;
    strtol(node->opdata, &endptr, 10);
	if (*endptr != '\0')
		return false;

	return true;
}

Calculator *cal_new(const char *opdata)
{
	Calculator *newptr = malloc(sizeof(Calculator));
	if (newptr == NULL)
	{
		fprintf(stderr, "malloc failed, %s\n", strerror(errno));
		return NULL;
	}

	memset(newptr, 0, sizeof(*newptr));
	strncpy(newptr->opdata, opdata, strnlen(opdata, sizeof(newptr->opdata)));
	newptr->tree_l= NULL;
	newptr->tree_r= NULL;
	return newptr;
}

int operator(const char *op, int a, int b)
{
	int result = 0;
	switch (op[0])
	{
		case '+':
			result = a + b;
			break;
		case '-':
			result = a - b;
			break;
		case 'x':
			result = a * b;
			break;
		case '/':
			result = a / b;
			break;
		default:
			break;
	}
	return result;
}

long str2num(const char *str)
{
	char *endptr;
	return strtol(str, &endptr, 10);
}

int cal_run(Calculator *root)
{
	int data_l;
	int data_r;

	if (!root) return 0;

	if ( cal_is_num(root) )
	{
		return str2num(root->opdata);
	} else 
	{
		data_l = cal_run(root->tree_l);
		data_r = cal_run(root->tree_r);
		return operator(root->opdata, data_l, data_r);
	}
}

int main(int argc, const char *argv[])
{
	do {
		/**
		 * such as:
		 *
		 *       5
		 *      / \
		 *     3   9 
		 *    / \   \
		 *   4   6   7
		 */
		Node *root  = node_new(5);
		root->left  = node_new(3); 
		root->right = node_new(9); 

		root->left->left  = node_new(4); 
		root->left->right = node_new(6); 
		root->right->right = node_new(7); 

		printf("tree_size = %d, tree_leaves = %d\n", 
				tree_size(root),
				tree_leaves(root));
	}while(0);

	do {

		/** 
		 * such as: 1 + 2x3 +  4x(3-1) == 15
		 *
		 *           +
		 *          / \
		 *         /   \
		 *        1     +
		 *             / \
		 *            /   \
		 *           x     x
		 *          /\     /\
		 *         /  \   /  \
		 *        2    3 4    -
		 *                   / \
		 *                  /   \
		 *                 3     1
		 */
		Calculator *root = cal_new("+");

		root->tree_l = cal_new("1");
		root->tree_r = cal_new("+");

		root->tree_r->tree_l = cal_new("x");
		root->tree_r->tree_r = cal_new("x");

		root->tree_r->tree_l->tree_l = cal_new("2");
		root->tree_r->tree_l->tree_r = cal_new("3");

		root->tree_r->tree_r->tree_l = cal_new("4");
		root->tree_r->tree_r->tree_r = cal_new("-");

		root->tree_r->tree_r->tree_r->tree_l = cal_new("3");
		root->tree_r->tree_r->tree_r->tree_r = cal_new("1");

		printf("calulator = %d\n", cal_run(root));
	}while(0);

    return 0;
}
```

## 平衡二叉树

- 哔哩哔哩-平衡二叉树(AVL树): https://www.bilibili.com/video/BV1tZ421q72h/?spm_id_from=333.788&vd_source=e9d237b5829f6446d4de63d10249c941
- 平衡二叉树简称AVL树，它是在二叉搜索树的基础上进一步优化得来。
- 平衡二叉树性质：首先它是二叉搜索树，在这个基础上，所有节点的(左子树高度 - 右子树高度)的绝对值小于等于1 —— 平衡因子(-1, 0, 1)。这样就能尽可能让树的分支保持均衡，不会出现一边倒的情况。
- 平衡二叉树的查找、插入、构建、删除过程和二叉搜索树是一样的，只是失衡的时候需要调整。即每次插入或者删除之后都需要检查一下是否存在失衡，如果失衡了就需要调整，怎么调整？用了基本的旋转操作（左旋、右旋）。

## 红黑树的原理

- 【哔哩哔哩：# 红黑树 - 定义, 插入, 构建】: https://www.bilibili.com/video/BV1Xm421x7Lg/?spm_id_from=333.337.search-card.all.click&vd_source=e9d237b5829f6446d4de63d10249c941
- 红黑树出现之前，是平衡二叉树，目的是为了防止二叉树的某个分支过长而另一个分支过短。
- 红黑树其实就是一种平衡二叉树。红黑树中，所有的子树都小于根，所有的右子树都大于根。并在此基础上，给每个节点打了一个颜色标记，根和叶子(NULL)都是黑色，同时所有红色节点的左右孩子节点都必须是黑色，换句话说，就是从上到下，不能出现连续的两个红色节点；另外，任何一个节点到叶子节点所有路径黑节点相同。
- 上面的性质口诀：左根右，根叶黑，不红红，黑路同。
- AVL树在查询上更高效，红黑树在插入和删除更高效。
- 红黑树的插入操作中，插入节点默认为红色，如果上述性质被破坏，则需要旋转+变色。

## 图-基本概念

- 典型应用：导航地图

---

- 图的基本概念: https://www.bilibili.com/video/BV1ET421673Y
- 定义-顶点集: V={A,B,C,D,E,F,D}
- 定义-边集: E={(A,B), (A,C), (A,D), (C,E), (B,F), (D,F), (E,D)}
- 定义-图: G=(V,E)
- 无向图: 顶点之间的连线没有方向。
- 无向图顶点的度: 跟该顶点产生交集的边，就称为顶点的度。
- 有向图顶点的度: 分为入度和出度，入度就是指向该顶点的连线的数量，比如多少箭头指向A顶点，使用ID(A)表示；出度是该顶点为起点的有向边数量，即多少个箭头从该顶点向外发射，使用OD(A)表示；就是有向图顶点的度=入度和出度之和。

## 图-邻接矩阵和邻接链表对比


- 对于稀疏图，使用邻接链表比较好；对于稠密图，使用邻接矩阵比较好。因为稀疏图对应的矩阵中，大量的格子都是0，所以这些0会占用大量不必要的空间。而稠密图中如果使用链表，由于链表节点中除了指针域，还有繁杂的数据域，相比矩阵会占用更多空间。
- 检查一条边是否存在时，对于邻接矩阵只需检查对应的元素位置是否为1即可，时间复杂度为O(1)；对于邻接链表，我们必须遍历A这条链表，找到指定的节点是否存在，才能判断边是否存在，时间复杂度为O(V)。
- 添加一条边，对于邻接矩阵，直接找到对应的行和列的元素，改成1即可，时间复杂度为O(1)；对于邻接链表，如果使用头插法，同样时间复杂度为O(1)。
- 删除一条边，对于邻接矩阵，把对应的元素设置为0即可，时间复杂度为O(1)；对于邻接链表，我们必须找到链表对应的节点，时间复杂度为O(V)。
- 添加一个顶点，对于邻接矩阵，则必须改变原有的矩阵大小，并且需要将原有矩阵的内容复制到新的矩阵中，时间复杂度为O(V)；对于邻接链表，只需在顶点表中，追加一个元素即可，时间复杂度为O(1)。

# 哈希表

## 哈希表-基本原理

- 在一个数组中查找某个值是否存在，通常用循环遍历来查找。缺点是在大量数据时效率低下。
- 哈希表版本01：最简单的哈希思想：一个大数组中，每个值直接是数组索引，比如查询60这个值是否存在，直接访问60这个索引下的值是否大于0，大于0表示数组中有60这个值。在初始化哈希表时，把数组里面的值都记录到哈希表对应的索引下，比如数组中60这个值出现了3次，那么在哈希表中对应的index=60这个索引下，统计为3。
- 哈希表版本02：如果key值不是一个0到100的整数，而是其他信息比如字符串等信息。对于那种 uint32_t 类型的整数，我们不可能初始化一个4GB大小的数组。因此我们会根据实际取值范围，定一个小的哈希表，比如下:

```cpp
#define MAX_TABLE_LEN 100
int get_table_index(int key) { return key % MAX_TABLE_LEN; }
```

- 但是上面这种的实际取值范围不能超过100，否则就导致重叠，比如 key=23 和 key=123 会得到同一个下标值（index=23）。这就是哈希冲突。


## 哈希表-哈希冲突

- HashMap是怎么解决哈希冲突的？其采用了链式寻址法，就是比如在index=6位置冲突了，该位置已经有键值对了，新来的键值对以单向链表的方式存放在其后面。HashMap通过【链式寻址法 + 红黑树】来解决哈希冲突问题，其中红黑树是为了优化Hash表链表过长导致的时间复杂度增加的问题。当链表长度大于等于8，并且哈希表的容量大于64的时候，再次向链表中添加元素的时候，就会触发链表向红黑树的转化。
