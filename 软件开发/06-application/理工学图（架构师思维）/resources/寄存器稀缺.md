

👉 寄存器稀缺结果：寄存器不够用 → 频繁访问内存



寄存器富集：变量放不下(超过寄存器数量) → spill 到内存



❗特点

- 某段代码：
    - 同时活跃变量很多（live set 很大）
- 超过寄存器数量

👉 结果：变量放不下 → spill 到内存

🔥 本质问题
> 👉 **软件层面：对“高速资源”的需求过高**



### 情况 1：供给不足（寄存器稀缺）

寄存器少 → 即使代码简单也会 spill

---

### 情况 2：需求过高（寄存器富集）

变量太多 → 即使寄存器不少也会 spill

👉本质目标：**减少同一时刻“活跃变量（live variables）”数量**


❌ 错误写法（典型寄存器富集）：
```cpp
// 案例1
int a = f1();  
int b = f2();  
int c = f3();  
int d = f4();  
int res = a + b + c + d;

// 案例2
for (int i = 0; i < N; i++) {
    int a = ...
    int b = ...
    int c = ...
    int d = ...
    int e = ...
    sum += a + b + c + d + e;
}
```

✅ 优化：
```cpp
int res = 0;
res += f1();
res += f2();
res += f3();
res += f4();


for (int i = 0; i < N; i++) {
    int tmp = 0;
    tmp += ...;  // a
    tmp += ...;  // b
    tmp += ...;  // c
    tmp += ...;  // d
    tmp += ...;  // e
    sum += tmp;
}

// 利用作用域（C/C++很少人用好这个）
{
    int a = ...
    use(a);
} // a 生命周期结束,👉 强制编译器缩短 live range
{
    int b = ...
    use(b);
}
```

👉 效果：
- 每个中间值“用完即死”
- 几乎不占寄存器

---

### 👉 本质统一：

> ❗**供需失衡 → 高速路径失效 → 退化到慢路径**
