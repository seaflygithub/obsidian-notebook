# VT-D BIOS固件

玩家反馈：玩三角洲老师弹窗要开VT-D，开了它但DMA板子容易被检测三天，软件过弹窗风险太大了还不如玩内存。

方案：让写固件的人研究一下主板BIOS的固件，把VT-D开启后的映射页伪造一个假的，然后腾讯反作弊就会去读假的，你板子就安全了也不会弹VT-D提示了。

可行：修改主板 BIOS 固件映射表（测试稳定可行）

B站UP：pc主板售后

VT-D的原理：通俗解释就是vtd可以监视设备在干什么事情

修改BIOS固件开启vtd过检测免费思路教程：

（1）vtd是intel的虚拟化技术，amd是iommu，可以理解为同一个东西，

（2）DMAR表：ACPI（高级配置与电源管理接口）中的一个表，用于描述DMA重映射硬件单元（如IOMMU）的信息，系统固件在启动时会将这个表提供给OS。

思路：修改BIOS固件中的DMAR表并固定，删除对应的设备表示不启用，使其初始化就不包含某个硬件（比如伪装的DMA）

教程开始：

（1）提取固件（方便查找已经生成的表）

（2）定位和解析ACPI表（包括DMAR），使用工具如 UEFITool 或 ACPI Table Browser 来查看和提取ACPI表，在UEFITool 中打开固件镜像，去搜索 “DMAR” 表，找到DMAR表之后，可以将其导出为bin或者scr文件。

（3）修改或替换DMAR表

预设的假表：需要手写一个假表，这个表应符合ACPI规范，并且包含必要的VT-d信息，如重映射结构、设备范围等，为了避免折腾麻烦，尽量基于原信息修改，替换表如果写的不好会出现莫名其妙的问题。

修改方法：

方法一：使用二进制编辑器直接修改固件镜像中的DMAR表区域，需要知道确切的偏移量。

方法二：使用 UEFITool 修改整个DMAR表，步骤如下： （1）在UEFITool中找到DMAR表，’DMAR‘ 头 + 长度64字节，保留域+硬件单元基本信息，每个bios都不一样，所以表并不通用。

（2）找到你想移除的设备把数值全改0，但是如果添加需要有个校验和计算，这里提供一个python代码：

```python
data = open("dmar_origin.bin", "rb").read()
checksum = (256-sum(data) & 0xFF) % 256     #ACPI校验和算法
data = bytearray(data)
data[9] = checksum                          #校验和偏移
open("dmar_hack.bin", "wb").write(data)
```
