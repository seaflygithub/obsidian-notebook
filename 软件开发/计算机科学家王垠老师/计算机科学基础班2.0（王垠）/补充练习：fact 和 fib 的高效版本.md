# fact 的高效版本

我们知道，普通递归版本的 fact 问题在于，它会把中间需要乘的结果全都保存起来，直到遇到 base case，才回去一个个的乘起来。现在我们实现一个效率高一些的 fact 函数，同样要用递归，然而它却不需要保存中间那些数。



你可以从下面的手算过程得到灵感。如果要计算 fact(5)，可以写出这样一个序列：
- 5, 1
- 4, 5
- 3, 20
- 2, 60
- 1, 120
- 0, 返回 120

观察一下，每一行有两个数。第一个数起先是 n，逐渐递减，就像递归的参数一样。第二个数最开头是 1，之后每次都变成上一行的两个数的乘积。当第一个数变成 0 的时候，第二个数就是 fact(n) 的结果了。



你可以根据这个过程写出一个有两个参数的递归函数，名字叫 fact1。fact1 的参数会实现以上的手算序列。然后在新的 fact 函数里面调用 fact1 ，这样就得到一个高效版本的 fact。



注意：请不要使用没有教过的内容，比如循环语句，赋值操作等。
