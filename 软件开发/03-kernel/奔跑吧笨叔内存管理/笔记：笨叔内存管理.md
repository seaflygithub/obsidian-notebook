

# 实验环境


VMware + ubuntu + qemu

## 编译运行busybox最小系统

```bash
sudo apt-get install -y qemu \
        libncurses5-dev \
        gcc-arm-linux-gnueabi \
        build-essential \
        gdb-arm-none-eabi \
        gdb-multiarch \
        gcc-aarch64-linux-gnu \
        eclipse-cdt 
git clone https://github.com/figozhang/runninglinuxkernel_4.0.git

cd runninglinuxkernel_4.0
# 该目录下有几个最小的文件系统: _install_arm32, _install_arm64, ...
# 创建基础设备节点:
sudo mknod _install_arm32/dev/console c 5 1

export ARCH=arm
export CROSS_COMPILE=arm-linux-gnueabi-
#export CROSS_COMPILE=arm-none-eabi-
make vexpress_defconfig

# 查看内核配置菜单: System Type >> ARM Ltd. Versatile Express family
make menuconfig
make bzImage -j4
make dtbs

# 之后就可以用qemu启动内核:
qemu-system-arm -nographic \
        -M vexpress-a9 -m 1024M \
        -kernel arch/arm/boot/zImage \
        -append "rdinit=/linuxrc console=ttyAMA0 loglevel=8" \
        -dtb arch/arm/boot/dts/vexpress-v2p-ca9.dtb
```

启动虚拟机:
```bash
cd runninglinuxkernel_4.0
sh run.sh arm32

# 在qemu里，执行如下操作:
cd /mnt
install xxx.ko
./appelf

# 最后想要快速结束qemu, 先按下Ctrl+A, 再快速按下x
```

## qemu配合gdb单步调试

1、在原来的qemu命令行末尾，加上额外两个选项：
```bash
qemu-system-arm -nographic \
        -M vexpress-a9 -m 1024M \
        -kernel arch/arm/boot/zImage \
        -append "rdinit=/linuxrc console=ttyAMA0 loglevel=8" \
```

## MMU职能

![[image_20240919203532.png]]

页索引占用12bits，所以换算下来页索引占用大小为4KB，
比如这里有3个虚拟地址需要转换，且三个虚拟地址的L1索引和L2索引相同，仅页索引不同，
虚拟地址1：0x40000110
虚拟地址2：0x40000220
虚拟地址3：0x40000330
虚拟地址4：0x40000FF0
上面这四个虚拟地址，都没有超出4KB范围（低12位），所以他们都在一个页里。

MMU访问页表这个行为是由硬件实现的，
但页表的创建和填充是由Linux内核来维护。
