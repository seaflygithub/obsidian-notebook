


# github笔记-密码验证

github remote: Support for password authentication was removed on August 13, 2021.

GitHub removed password authentication for Git operations on August 13, 2021. Instead, you must use **Personal Access Tokens** (PAT) or **SSH keys** for authentication.

**Solution 1: Using Personal Access Tokens (PAT)**

```txt
- Log in to your GitHub account and go to Settings > Developer settings > Personal access tokens.
- Provide a description, set an expiration date(default 30 days), and select the required scopes (e.g., repo for repository access). For Personal access tokens (classic).
- Generate the token and copy it (you won’t see it again).
- Use the token in place of your password(if in China, you need to use VPN mothod): git clone <[<https://<your_token>@github.com/>](https://%3Cyour_token%3E@github.com/)><username>/<repo>.git
- And then you needn’t authentication for remote operations.
```


# github笔记-覆盖提交

**覆盖 / 合并多个历史提交（变基）**

如果想把最近 N 次提交合并为一次（覆盖成一个提交），用 `git rebase -i`（交互式变基）：

**步骤：**

1、确定要合并的提交数量，比如最近 3 次：

```cpp
git rebase -i HEAD~3
```

2、弹出编辑器，显示 3 条提交记录，格式如下：

```cpp
pick a1b2c3d 第一次提交备注
pick d4e5f6g 第二次提交备注
pick g7h8i9j 第三次提交备注
```

3、修改除第一条外的 `pick` 为 `squash`（或简写 `s`），表示合并到上一条：

```cpp
pick a1b2c3d 第一次提交备注
s d4e5f6g 第二次提交备注
s g7h8i9j 第三次提交备注
```

1. 保存退出编辑器，会弹出新编辑器让你编辑合并后的提交备注；
2. 保存后，3 次提交会被合并为 1 次（覆盖原有 3 条记录）；
3. 若已推送到远程，强制推送： `git push -f`
