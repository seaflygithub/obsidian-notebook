## zynq-fan转速原理


下面仅仅是分析了zynq-ps 端的代码分析得到的原理图，实际情况是由 zynq-pl 端采集or控制风扇转速，所以会丢失具体控制的内部细节，比如占空比是怎么设置的，以及如何获取风扇high_time的。这些细节在 zynq-fan 工程里，PS软件层面是没法获知的。好在本项目是单片机，所有风扇转速的采集和控制都需要单片机代码来实现，所以可以探究其控制细节。

分析之前 zynq-fan 风扇转速计算代码:
```cpp
unsigned short get_fan_rpm(unsigned int high_time)
{
	unsigned short fan_rpm;

	high_time &= 0x00ffff00;
	if( high_time == 0 )
		return 0;

	fan_rpm = (unsigned short)((unsigned long long)60*g_fan_detect_clk*1000000/(high_time*4));
	if( fan_rpm > 12000 )
		fan_rpm = 0;

	return fan_rpm;
}
```

其中风扇转速脉冲采集时钟频率是 5MHz，即代码中的乘以 1000000，high_time 应该是指高电平持续的tick数，而乘以4则表示风扇转一圈消耗的总tick数。

STM32 的定时器 PWM 模式分为**PWM1**和**PWM2**两种，它们的核心区别在于**电平翻转的条件相反**。


1、直接把 TIM3_PWM_Init 函数实现代码贴给AI，让AI分析并注释，你只要看懂为什么这么用就行。

TIMx_CNT: 当前计数器的值，实时累加的计数值，从 0 开始向上计数；
TIMx_ARR: 计数器的最大值，CNT 累加到 ARR 后会自动清零，重新从 0 开始计数，形成循环；
TIMx_CCR1: 通道1的比较值，是 PWM 电平切换的 “阈值”，CCR2、CCR3、CCR4 同理。


```cpp
void Adjust_Fan_Speed(u8 speed_value)
{
	static u8 old_fan_speed = 0xff;
	
	if( speed_value > 100 )
		return;
	
	if( speed_value == old_fan_speed )
		return;
	
	// 根据百分比设置占空比(计数器比较寄存器) CCR1,CCR2,...
	TIM_SetCompare1(TIM3, 30*(100-speed_value));

	old_fan_speed = speed_value;
	Fan_Speed_Now = speed_value;
}
```



ClockDivision=DIV4 相当于秒表的 “读数精度”，比如原本能精确到 1 秒，现在只能精确到 4 秒（仅影响 “读数”，不影响秒表本身的跳动速度）。

<font color=blue>ClockDivision(CKD) 仅影响读数，这里的读数是读什么数？</font>

CKD只影响对外部信号的采样频率，不影响计数器计数精度。

<font color=blue>只有用到输入捕获 / 输出比较时，才需要根据外部信号的频率调整 CKD 来优化采样精度。 为什么需要根据外部信号频率调整CKD来优化采样精度？若不通过CKD分频，那么采样精度不是更高么？分频了之后采样精度变低了，那还有CKD配置的必要么？</font>

你提出的这个问题非常关键 —— 既然 CKD 分频会降低采样时钟频率、看似降低精度，为什么还要配置它？核心原因是：**采样精度≠采样可靠性**，CKD 的作用不是追求 “极致精度”，而是在 “精度” 和 “抗干扰 / 稳定性” 之间找平衡，适配不同频率的外部信号。

在实际工程中，外部信号（如传感器、编码器脉冲）往往带有噪声、毛刺，过高的采样频率会把这些干扰误判为有效信号，反而导致采样结果完全错误。

<font color=red>比如我的采样精度是10秒，也就是我10秒才会采样一次，那么在10秒范围内，产生的任何毛刺干扰波形，我都不会搭理它。这就体现了低频采样的稳定性or抗干扰能力。如果目标有效信号是大于等于10秒产生电平变化，那么10秒采样间隔就是最优选择。</font>


## TIMER5-捕获脉冲


大多数风扇每转一圈就会产生2个完整周期的脉冲，转速越快，周期越小，反之越大。

对应的初始化函数: TIM5_Cap_Init, TIM4_Cap_Init, TIM8_Cap_Init

核心功能: 核心功能是配置 TIM5 的 4 个通道（CH1~CH4）为输入捕获模式，用于捕获外部脉冲信号的边沿。另外还有其他定时器捕获剩余风扇的转动脉冲。

![[Pasted image 20260106184450.png]]

CNT计数器频率 = 72MHz / 720 = 100kHz, 即tick频率是100kHz；
对风扇脉冲采样频率 = CNT频率 / DIV4 = 25kHz, 即对外部脉冲的采样频率为25kHz。

```cpp
static u16 Get_Capture_Tick(u16 first, u16 second)
{
	u16 value = 0;
	
	if( second >= first )
	{
		// 普通差值计算: 直接相减
		value = second - first;
	}
	else
	{
		// 计数器溢出情况下的差值计算
		value = second + (0xffff - first);
	}
	
	return value;
}
```

<font color=blue>请问这里统计出来的计数器差值 FAN1_CAPTURE_VAL，这是表示是统计半个脉冲所消耗的计数器差值吗？</font>

你这个理解**非常准确**，该差值表示的是 **两个相邻的、不同极性边沿之间的计数器差值**，更直观地说：它对应的是「一个脉冲的低电平持续时间」或「一个脉冲的高电平持续时间」，而非单个完整脉冲（上升沿到下降沿）的总时长。

<font color=blue>AVC DBTB1225B8F DC 48V 0.48A 这个型号的风扇，风扇每转动一圈，会输出几个完整脉冲信号？</font>

AVC DBTB1225B8F DC 48V 0.48A 风扇每转动一圈，通常会输出**2 个完整脉冲信号**（FG 信号，转速反馈信号）。这是 AVC 该系列 4 线 PWM 风扇的通用设计，其内部多为 4 极电机，电机每转一圈，霍尔传感器会感应到 2 次磁极变化，从而输出 2 个脉冲信号。
