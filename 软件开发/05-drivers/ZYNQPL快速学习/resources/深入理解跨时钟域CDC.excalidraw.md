# Excalidraw Data

## Text Elements
多bit跨域必须用协议
本质原因: 多bit信号 "不同步翻转"


典型例子: binary计数器
理想情况: 
    t0: 0111
    t1: 1000
真实硬件:
    t0: 0111
    t1: 0110
    t2: 0100
    t3: 0000
    t4: 1000

0111 → 0110 → 0100 → 0000 → 1000
👉 中间一堆“垃圾状态”（非法中间态）
👉 组合判断时就容易误判（毛刺）


多bit信号：
(1)理想情况: 一次跳变 = 多bit同时变化
(2)现实情况: 一次跳变 = 多bit不同时变化


👉 标准工程说法：
Gray Code 不能消除问题，
它只是避免多bit同时变化带来的非法状态。

二级同步器也不能消除亚稳态，
它只是将亚稳态传播的概率降低到极低。

Gray Code + 二级同步器 + 时序约束
可以让系统达到工程上“等价于可靠”的程度，
但从理论上讲，永远不可能100%消除错误。


你可以把 CDC 看成：
不是在“避免错误”
而是在“压低错误概率” ^G4BSJIod

"非法中间态" ^oLnFZCi1

二级同步器（async --> FF1 --> FF2）
👉 “我不相信你这一拍的结果，我给你一整拍时间去‘恢复’，然后下一拍再采你。”


第1拍（危险拍）
FF1 采样异步信号
可能：
正常 0/1 ✅
进入亚稳态 ⚠️
第2拍（关键拍）
FF2 采 FF1 的输出

此时有两种情况：
✅ 情况1：FF1 已经恢复
输出稳定 0 或 1
FF2 采到正确值
⚠️ 情况2：FF1 还在亚稳态（极小概率）
FF2 也可能被污染
 ^bMWTmCRC

既然二级同步器这么好用，为什么多增加几级？比如三级同步器、四级同步器等等 ^NYXRns58

4️⃣ 收益递减（关键）
这是最重要的工程思维：
👉 2级已经把错误率压到极低
再往上：从 10⁻¹² → 10⁻¹⁸ → 10⁻²⁴

但现实中：系统其他地方早就先坏了（比如 SEU、EMI、软件 bug）
👉 同步器已经不是瓶颈
👉 CDC设计的目标不是“绝对正确”，而是“系统级可接受的错误概率 + 正确的架构约束”


推导一个真实项目的 MTBF，算出“到底用2级还是3级”
800M以上推荐多少级同步器，可以直接问AI



👉 风险窗口其实不是一个“点”，而是一小段时间：
      |<--- setup --->| clk edge |<--- hold --->|
如果输入在这个窗口变化：就可能进入亚稳态
