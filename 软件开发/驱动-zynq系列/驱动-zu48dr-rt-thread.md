
从 rtthread_startup 函数开始，它里面执行了很多子函数，


**Vitis裸机代码是从哪里跳转到main函数的？**

通过修改BSP相关位置，给main函数传递参数，最终确认到是 standalone 顶层源码跳转到 main 函数执行的。

![[Pasted image 20251222181802.png]]


```cpp
// 获得经验:

/*************************************************************************

	stp     x29, x30, [x0, #-16]!
	上述语句翻译: (char *)ptr -= 16, x29=ptr[0], x30=ptr[1]
	即栈指针先降两格, 然后按照数组索引方式把两个寄存器的值赋值给数组的两个元素;
	
	ldp     x19, x20, [x1], #16
	上述语句翻译: x19=ptr[0], x20=ptr[1], (char *)ptr += 16
	即先按照数组索引方式给两个寄存器逐个赋值, 再让栈指针升两格;
	
	根据上面的压栈和出栈的过程，可以知道:
	    (1) 满栈(sp指向位置是有数据的)、降栈(push操作让地址递减)
	    (2) 根据上述探究结果,就能知道 aarch64 的栈是什么样子了

**************************************************************************/
```
