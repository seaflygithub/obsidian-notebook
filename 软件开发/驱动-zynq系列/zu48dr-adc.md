

# 时钟体系


## LMK04832+ADF4377

---

<font color=blue>经常在工程里听说这两个时钟相关的信息，比如 zu48dr 里，04832+4377。04832和4377好像是他们讨论的和时钟架构里相关的模块。LMK04832+ADF4377。
</font>

这是高速 FPGA / RFSoC / JESD204 系统里非常经典的：
时钟树（Clock Tree）芯片，也叫 Jitter Cleaner（抖动清理器）

```txt
晶振
   ↓
LMK04832
   ↓
FPGA / ADC / DAC / JESD
```

FPGA 世界里的“总指挥”，它负责：
- 时钟倍频
- 分频
- 清理抖动
- 相位同步
- SYSREF 生成
- 多路时钟分发

**为什么 RFSoC 必须有这种芯片**

因为高速 ADC/DAC极度依赖时钟质量，例如 14bit ADC，如果时钟抖动(100fs >> 300fs)，SNR 都可能明显下降。




---

ADI ADF4377 RF PLL Synthesizer 是一颗非常典型的 **高频宽带 PLL + VCO 集成芯片**，在 RF、雷达、通信、FPGA外部时钟系统里非常常见。它是一个“超高频可调本振 + PLL 合成器”。给它一个参考时钟，它可以合成几十 MHz ~ 几 GHz 的干净射频时钟。

```txt
晶振
  ↓
LMK04832（低抖动时钟树）
  ↓
ADF4377（高频本振/合成）
  ↓
ADC / DAC / RF 前端
  ↓
FPGA / RFSoC
```


**它和 LMK04832 最大区别**

| 芯片       | 本质                                  | 作用            |
| -------- | ----------------------------------- | ------------- |
| LMK04832 | Clock Jitter Cleaner + Distribution | “把时钟变干净 + 分发” |
| ADF4377  | Wideband RF PLL/VCO                 | “生成超高频时钟”     |

可以理解为：
LMK = 清洁工 + 分发中心
ADF = 高速发动机


**ADF4377 内部结构**

它本质是：Phase Locked Loop + VCO + 分频器

```txt
Reference Clock
     ↓
Phase Detector
     ↓
Charge Pump
     ↓
Loop Filter
     ↓
VCO (GHz级)
     ↓
Output Divider
     ↓
RF Output
     ↑
Feedback
```


它能干到什么频率？

ADF4377 特点：
- 输出范围：大约 **~62 MHz 到 ~32 GHz（分频后）**
- VCO 本体：通常在 GHz 级（比如 6~12 GHz）

所以它可以：
```
生成雷达本振 / LO / 高速采样时钟
```


**场景：雷达 / 通信**

```txt
基带 FPGA
   ↓
DAC 采样 491.52MHz
   ↓
上变频 LO = 3~10 GHz（ADF4377）
   ↓
射频输出
```

**在PLL里，VCO处于什么位置**

```txt
参考时钟 → 相位比较器 → 环路滤波 → VCO → 输出
                         ↑              ↓
                         └────反馈分频───┘
```

**本振是什么意思？射频又是什么意思？**

RF = Radio Frequency（射频）

它本质是：**用电磁波来传输信息的高频信号**

低频（基带）：0 ~ 10 MHz
射频（RF）：把数据“搬到高频载波上”，比如 2.4 GHz WiFi

RF不是“另一种信号”，而是：👉 “被搬到高频上的信息”



LO = Local Oscillator（本地振荡器）
它本质是：
> 👉 一个“固定频率的纯净正弦波源”


本振到底用来干嘛？它的作用只有一个核心：
> **把信号“搬家”（频率转换）**

```txt
RF信号 + LO信号 → Mixer（混频器） → 新频率信号
```


**一个真实工程例子（WiFi）**


发送端：
```txt
数据（100 MHz以内）
   ↓
调制
   ↓
混频 + LO（2.4 GHz）
   ↓
射频信号（2.4 GHz WiFi）
```

接收端：
```txt
2.4 GHz RF信号
   ↓
LO（2.4 GHz）
   ↓
下变频
   ↓
基带信号
```



| 名称     | 本质        | 作用                |
| ------ | --------- | ----------------- |
| RF（射频） | 携带信息的高频信号 | “数据在飞”            |
| LO（本振） | 固定纯净频率源   | 只复杂变频(上变频 or 下变频) |

可以简化理解:
发射端: 基带 → +LO → RF
接收端: RF → -LO → 基带

<font color=blue>上述信号还提到了 IQ 的概念，IQ是什么？在信号中起到什么作用？为什么要有IQ这个技术？
</font>

你可以把它理解成一句话：
> **IQ = 把一个高频信号拆成“两个正交的低频分量”，用来完整描述它的幅度和相位**


<font color=blue>既然ADC采样量化后的数据只有幅值信息，那么为什么还要有IQ这个技术？
</font>

❗ 单通道 ADC 只能记录“**波的高度**”，但 RF 通信需要的是“**波的振幅 + 相位 + 频率方向**”，IQ 是为了把这些信息完整保留下来。


<font color=blue>那如果我需要把信号绘制到界面上(坐标系图），我现在有一组IQ数据（为了简化探究，这里只有8个IQ数据），我要如何利用这IQ数据，把它绘制到 x,y 坐标系上。如何把它绘制到时域图上。
</font>

IQ 数据本身**已经是坐标系点了**：
> ✔ I = x轴  
> ✔ Q = y轴
