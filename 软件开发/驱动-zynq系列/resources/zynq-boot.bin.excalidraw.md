环境准备：
        在编译Linux机中按照vivado环境（交叉编译环境），执行以下脚本
source /Xilinx_2018_03/Vivado/2018.3/settings64.sh
export CROSS_COMPILE=arm-linux-gnueabihf-
export ARCH=arm

//保存当前配置（保存当前.config文件为defconfig）
make savedefconfig




操作步骤：
1、制作FSBL
按照提示生成fsbl工程（注意选择打印串口），编译出fsbl.elf文件备用

2、制作devicetree
首先配置SDK
1）打开SDK。
2）选择：Xilinx-> Repositories
3）在Local Repositories中点击New，将在github中下载的文件夹路径添加进去。
4）点击OK，等待编译完成，大概需要2分钟。
然后使用Device Tree Generator生成设备树，注意bootargs可以设置成：
console=ttyPS0,115200n8 earlyprintk rootwait
然后把自动生成的整个文件夹导入到linux编译机中去，制作Makefile，如下所示：

# Makefile generated by Xilinx.
all:
        $(RM) -rf system-top.dtb
        gcc -E -nostdinc -undef -D__DTS__ -x assembler-with-cpp -o system-top.dts.tmp system-top.dts
        dtc -I dts -O dtb -o system-top.dtb system-top.dts.tmp
        $(RM) -rf *.tmp
放入同一文件夹，编译成dtb文件，把该dtb文件导出来改名成devicetree.dtb备用


3、制作uboot
make distclean
make xilinx_zynq_54suo_defconfig
export DEVICE_TREE="device_tree_bsp_0/system-top"        
//也可直接修改xilinx_zynq_54suo_defconfig文件中相关位置，
//目的是找到该devicetree,make编译成功后导出该u-boot.elf备用


4、编译zynq linux
编译zynq linux源码（根据要求自行裁剪），生成uImage文件备用
make mrproper
make xilinx_zynq_defconfig
make UIMAGE_LOADADDR=0x8000 uImage
生成文件在以下目录：../arch/arm/boot/uImage

5、制作文件系统
dd if=uramdisk.image.gz of=ramdisk.image.gz skip=16 bs=4        //去掉64字节头文件
gunzip ramdisk.image.gz                                                //解压缩
mount -o loop ramdisk.image /mnt                                //挂载到本地进行编辑
umount /mnt                                                        //卸载
gzip ramdisk.image                                                //重新压缩
mkimage -A arm -T ramdisk -C gzip -d ramdisk.image.gz uramdisk.image.gz                //加上64字节头文件

6、准备zynq linux运行文件
把前面生成的文件fsbl.elf, system_wrapper.bit, u-boot.elf, devicetree.dtb，uImage，uramdisk.image.gz合成Boot.bin文件，合成参数如下：
//arch = zynq; split = false; format = BIN
the_ROM_image:
{
        [bootloader]E:\Xilinx_SDK\41suo\41suo_zynq\bootbin\fsbl.elf
        E:\Xilinx_SDK\41suo\41suo_zynq\bootbin\system_wrapper.bit
        E:\Xilinx_SDK\41suo\41suo_zynq\bootbin\u-boot.elf
        [load = 0x1800000]E:\Xilinx_SDK\41suo\41suo_zynq\bootbin\devicetree.dtb
        [load = 0x1000000]E:\Xilinx_SDK\41suo\41suo_zynq\bootbin\uramdisk.image.gz
        [load = 0x1900000]E:\Xilinx_SDK\41suo\41suo_zynq\bootbin\uImage
}

7、启动linux
上电启动，进入到uboot界面后，输入以下命令：
setenv boot_targets zynqlinux
setenv bootcmd_zynqlinux bootm 0x1900000 0x1000000 0x1800000
saveenv
reset

8、Linux启动成功。


注：在uboot下查看环境变量命令为：pri
可配合uboot设置和环境变量确定uboot脚本启动顺序，确保uboot从qspi flash卡把linux所需各文件导入到ddr中并运行，本例脚本启动位置为：bootcmd。


由于在zynq Linux中没有加入板载动态链接时的运行环境，编译运行代码时需要静态编译，如：
arm-linux-gnueabihf-gcc helloworld.c -o helloworld -static
